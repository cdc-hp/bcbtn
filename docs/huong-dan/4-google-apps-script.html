Hướng dẫn triển khai Google Apps Script

Máy chủ phụ + trang GitHub Pages cho các xã nộp báo cáo

Tài liệu 4/5 — Ứng dụng Giám sát dịch bệnh

1. Vai trò của Google Apps Script trong hệ thống

Máy chủ chính chỉ nghe trong LAN nội bộ CDC nên các xã ở xa không vào thẳng được. Google Apps Script đóng vai trò "cửa sổ online" công khai: xã nộp Excel qua đó, script tự chuyển tiếp thẳng vào máy chủ chính nếu có thể, hoặc lưu tạm vào Google Sheet/Drive (không chia sẻ công khai) khi máy chủ chính tạm thời không phản hồi. Toàn bộ mã nguồn nằm ở google_apps_script/Code.gs trong repo.

2. Việc cần làm 1 lần khi thiết lập mới

2.1 Tạo project Apps Script

1
Vào script.google.com (bằng tài khoản Google của CDC) → New project.
2
Xoá nội dung file mặc định, dán toàn bộ nội dung google_apps_script/Code.gs từ repo vào.
3
Mở Project Settings (biểu tượng bánh răng bên trái) → tick "Show appsscript.json manifest file in editor" → dán nội dung google_apps_script/appsscript.json từ repo vào file đó (đảm bảo có sẵn khối "webapp" với executeAs: USER_DEPLOYING, access: ANYONE_ANONYMOUS — thiếu phần này sẽ không nhúng được vào GitHub Pages).
Kỹ thuật viên có thể dùng công cụ dòng lệnh clasp (clasp clone-script <scriptId>, clasp push, clasp deploy) để làm nhanh hơn thay vì copy/dán thủ công — xem thêm tài liệu clasp chính thức của Google nếu cần.

2.2 Cấu hình Script Properties

Vào Project Settings → Script Properties → Add script property, thêm:

Thuộc tínhBắt buộc?Ý nghĩa
SHARED_KEYCóKhoá bí mật các xã nhập khi nộp báo cáo. CDC tự đặt (chuỗi ký tự bất kỳ, đủ dài).
ROOT_FOLDER_IDKhôngGhi đè folder Drive lưu hàng chờ (mặc định script tự tạo folder mới trong Drive của tài khoản sở hữu nếu bỏ trống).
MAIN_SERVER_URLKhôngChỉ đặt nếu máy chủ chính đã có địa chỉ Internet thật (đã mở cổng — xem cảnh báo Mục 4). Ví dụ: https://cdc-haiphong.example.com:8765.
MAIN_SERVER_PASSWORDKhôngĐi kèm MAIN_SERVER_URL — mật khẩu dùng chung của máy chủ chính.

2.3 Deploy dưới dạng Web App

1
Bấm Deploy → New deployment.
2
Loại: Web app. Execute as: Me. Who has access: Anyone.
3
Bấm Deploy, cấp quyền khi được hỏi (chấp nhận cảnh báo "app chưa được Google xác minh" — đây là script của chính CDC).
4
Sao chép Web app URL dạng https://script.google.com/macros/s/XXXXXXXX/exec.
Lần deploy ĐẦU TIÊN có thể vẫn báo lỗi truy cập công khai dù đã chọn "Anyone" — vào lại Deploy → Manage deployments, bấm sửa (biểu tượng bút chì), xác nhận lại "Who has access: Anyone" rồi Save/Deploy một lần nữa để kích hoạt đúng.

3. Kết nối với GitHub Pages (link cố định cho các xã)

1
Mở file docs/config.js trong repo cdc-hp/bcbtn.
2
Sửa giá trị GAS_URL thành đúng Web app URL vừa sao chép ở Mục 2.3.
3
Commit & push thay đổi lên nhánh main — GitHub Pages tự build lại sau khoảng 1 phút.
4
Kiểm tra lại bằng cách mở https://cdc-hp.github.io/bcbtn/ — phải thấy đúng form nộp báo cáo hiện ra trong khung trang.
Chỉ cần sửa config.js khi tạo deployment mới (URL đổi). Nếu chỉ cập nhật code rồi Deploy lại đúng deployment cũ (Manage deployments → Edit → New version), URL giữ nguyên, không cần sửa gì trên GitHub Pages.

4. Mở máy chủ chính ra Internet (tuỳ chọn nâng cao)

Mặc định, mọi báo cáo nộp qua Apps Script đều lưu tạm vào Sheet/Drive rồi CDC đồng bộ bù thủ công. Nếu muốn báo cáo vào thẳng máy chủ chính ngay khi nộp (không cần đồng bộ bù), cần mở máy chủ chính ra Internet qua domain cdc-hp.io.vn — hướng dẫn từng bước đầy đủ (Cloudflare Tunnel, không cần port-forward): xem Tài liệu 5/5. Sau khi hoàn tất, chỉ cần đặt 2 Script Property ở Mục 2.2: MAIN_SERVER_URL = https://cdc-hp.io.vn và MAIN_SERVER_PASSWORD.

Đây là thay đổi có rủi ro bảo mật (máy chủ nhận request từ Internet công khai) — chỉ nên bật khi đã cân nhắc kỹ, và luôn đi kèm mật khẩu máy chủ (xem Tài liệu 5/5, Mục 5).

5. Vận hành hằng ngày

  • CDC vào /cdc/hang-doi trên máy chủ chính (hoặc tab Hàng đợi trên máy trạm) để xem báo cáo đã nộp, nhập vào CSDL.
  • Bấm "Đồng bộ máy chủ phụ" định kỳ (vài lần/ngày) để kéo các báo cáo đang ở trạng thái "chờ đồng bộ" từ Google Sheet/Drive vào hàng đợi chính — kể cả khi đã bật chuyển tiếp trực tiếp, vẫn nên làm để vét các lần chuyển tiếp lỡ thất bại.
  • Nếu đổi máy chủ chính (xem Tài liệu 2/5, Mục 6), nhớ cập nhật lại MAIN_SERVER_URL/MAIN_SERVER_PASSWORD trong Script Properties nếu đang dùng chuyển tiếp trực tiếp.
