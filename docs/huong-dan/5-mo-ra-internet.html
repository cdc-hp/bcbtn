Hướng dẫn mở máy chủ chính ra Internet

Cloudflare Tunnel — không cần port-forward, không cần IP tĩnh, không cần liên hệ nhà mạng

Tài liệu 5/5 — Ứng dụng Giám sát dịch bệnh

1. Vì sao chọn Cloudflare Tunnel

Cách thông thường (port-forward trên router) cần quyền quản trị thiết bị mạng — nhiều nơi (như CDC Hải Phòng, mạng do VNPT/nhà mạng quản lý) không có quyền này hoặc phải xin qua nhiều bước. Cloudflare Tunnel làm ngược lại: máy chủ tự kết nối ra ngoài tới Cloudflare (kết nối đi ra thì mạng nào cũng cho phép sẵn), Cloudflare nhận request từ domain rồi chuyển vào qua đúng đường kết nối đó — không cần mở cổng, không cần IP công khai cố định, không cần xin nhà mạng.

Vẫn cần domain cdc-hp.io.vn đã đăng ký (đã có) — chỉ khác cách trỏ domain.

2. Chuẩn bị

  • Tài khoản Cloudflare (miễn phí) — đăng ký tại cloudflare.com nếu chưa có.
  • Máy đang chạy chế độ Máy chủ (app GiamSatDichBenh-Server-Setup-*.exe) — cloudflared cài trên chính máy này.

3. Chuyển domain sang Cloudflare quản lý DNS

1
Đăng nhập dash.cloudflare.com → Add a site → gõ cdc-hp.io.vn.
2
Cloudflare tự quét các bản ghi DNS hiện có (kể cả bản ghi A đã thêm trước đó) — chọn gói Free, bấm Continue.
3
Cloudflare hiện ra 2 nameserver (dạng xxx.ns.cloudflare.com). Copy lại.
4
Quay lại trang quản lý domain (nơi đã đăng ký cdc-hp.io.vn) → tìm mục đổi Nameserver (khác với "DNS Manager" đã dùng lúc trước) → thay 2 nameserver cũ bằng 2 nameserver Cloudflare vừa copy.
5
Chờ Cloudflare xác nhận domain đã hoạt động (thường vài phút tới vài giờ, đôi khi tới 24 giờ) — Cloudflare tự gửi email báo khi xong.
Bản ghi A đã thêm ở bước trước (trỏ về 14.224.232.9) không còn cần thiết nữa sau bước này — Cloudflare Tunnel sẽ tự quản lý bản ghi DNS của domain theo cách khác (Mục 5), không dùng IP đó nữa.

4. Cài cloudflared trên máy chủ

1
Tải cloudflared cho Windows tại github.com/cloudflare/cloudflared/releases/latest (file cloudflared-windows-amd64.exe), đổi tên thành cloudflared.exe, đặt vào thư mục cố định vd C:\cloudflared\.
2
Đảm bảo máy chủ chính (app, chế độ Máy chủ) đang chạy, nghe cổng 8765 — kiểm tra ở tab Server.

5. Tạo Tunnel (làm trên trang web Cloudflare)

1
Vào one.dash.cloudflare.com → chọn tài khoản → mục Networks → Tunnels (tên mục có thể hơi khác nếu Cloudflare đổi giao diện, tìm chữ "Tunnels").
2
Bấm Create a tunnel → chọn loại Cloudflared → đặt tên vd cdc-hp-server.
3
Cloudflare hiện ra 1 lệnh cài đặt cho Windows, dạng:
cloudflared.exe service install eyJhIjoixxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
4
Mở PowerShell với quyền Administrator tại C:\cloudflared\, dán và chạy đúng lệnh đó (chuỗi dài sau service install là mã riêng của tunnel bạn vừa tạo — copy nguyên văn từ trang Cloudflare, không gõ tay).
5
Quay lại trang Cloudflare, tunnel chuyển trạng thái Connected (màu xanh) — nghĩa là máy chủ đã kết nối thành công tới Cloudflare.
6
Vẫn trong trang cấu hình tunnel đó, sang tab Public Hostname → Add a public hostname:
TrườngGiá trị
Subdomainđể trống
Domaincdc-hp.io.vn
TypeHTTP
URLlocalhost:8765
7
Lưu lại — Cloudflare tự tạo bản ghi DNS cần thiết, không cần vào lại DNS Manager thêm gì nữa.
8
Kiểm tra: mở trình duyệt trên máy khác (ngoài mạng CDC, vd dùng 4G) truy cập https://cdc-hp.io.vn/health — phải thấy phản hồi JSON (hoặc yêu cầu mật khẩu nếu máy chủ đã đặt), không phải lỗi kết nối.
Từ bước 4, cloudflared chạy nền như dịch vụ Windows — tự khởi động lại cùng máy, không cần mở cửa sổ PowerShell chờ sẵn.

6. Bảo mật

Bắt buộc đặt mật khẩu máy chủ (tab Server trong app) trước khi hoàn tất — đây là lớp xác thực duy nhất cho request đến từ Internet, vì Cloudflare Tunnel chỉ lo việc kết nối, không thay được việc xác thực của ứng dụng.

7. Cập nhật cấu hình sau khi đã chạy

7.1 Google Apps Script (chuyển tiếp trực tiếp)

script.google.com → project đang dùng → Project Settings → Script Properties:

Thuộc tínhGiá trị

7.2 Máy trạm quản trị ở xa

1
Mở app → "Kết nối máy chủ LAN".
2
Đổi "Địa chỉ máy chủ" thành https://cdc-hp.io.vn.
3
Bấm "Kiểm tra kết nối" để xác nhận trước khi Lưu.
